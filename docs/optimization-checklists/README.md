# Оптимизация: пакет чеклистов

Этот набор документов разбивает большие планы по улучшению проекта на короткие,
атомарные задачи в формате `todo`.

Каждая задача содержит:
- конкретные действия;
- список файлов и артефактов;
- проверяемый результат.

## Порядок выполнения

1. [Фаза 0 - Метрики и наблюдаемость](./phase-0-observability-checklist.md)
2. [Фаза 1 - Быстрые победы](./phase-1-quick-wins-checklist.md)
3. [Фаза 2 - Конкурентность и throughput](./phase-2-concurrency-checklist.md)
4. [Фаза 3 - Кэш и сеть](./phase-3-cache-network-checklist.md)
5. [Фаза 4 - Очистка legacy и dead code](./phase-4-legacy-cleanup-checklist.md)
6. [Фаза 5 - Гейты качества и защита от регрессий](./phase-5-quality-gates-checklist.md)
7. [Фаза 6 - Упрощение приложения](./phase-6-simplification-checklist.md)

## Правила выполнения

- Выполнять задачи последовательно по ID внутри одной фазы.
- Один PR = одна атомарная задача.
- Галочку ставить только после реальной проверки результата.
- Если меняется формат данных, нужно сразу описывать совместимость или миграцию.
