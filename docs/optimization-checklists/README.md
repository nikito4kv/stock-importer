# Оптимизация: Пакет Чеклистов

Этот набор документов разбивает план оптимизации на атомарные задачи формата `todo`.

Каждая задача содержит:
- конкретные действия;
- список файлов/артефактов;
- проверяемый результат (Definition of Done).

## Порядок Выполнения

1. [Фаза 0 — Метрики и наблюдаемость](./phase-0-observability-checklist.md)
2. [Фаза 1 — Быстрые победы (I/O, polling, hot-path)](./phase-1-quick-wins-checklist.md)
3. [Фаза 2 — Конкурентность и throughput](./phase-2-concurrency-checklist.md)
4. [Фаза 3 — Кэш и сеть](./phase-3-cache-network-checklist.md)
5. [Фаза 4 — Очистка legacy и dead code](./phase-4-legacy-cleanup-checklist.md)
6. [Фаза 5 — Гейты качества и защита от регрессий](./phase-5-quality-gates-checklist.md)

## Правила Выполнения

- Выполнять задачи последовательно по ID внутри фазы.
- Один PR = одна атомарная задача.
- Пока задача не имеет подтвержденного результата из раздела "Проверка", галочку не ставить.
- При изменении формата данных всегда добавлять обратную совместимость или миграцию.
