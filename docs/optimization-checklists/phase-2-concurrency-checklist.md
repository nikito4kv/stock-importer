# Фаза 2: Конкурентность И Throughput

Цель фазы: ускорить run за счет контролируемого параллелизма без рисков thread-safety.

## P2-01 [x] Формализовать режимы конкурентности

Действия:
1. Ввести явный enum/режим исполнения: `storyblocks_safe`, `free_images_parallel`, `mixed_safe`.
2. На старте run вычислять режим из набора провайдеров.
3. Привязать лимиты worker pools к режиму.

Артефакты:
- `pipeline/media.py`
- `providers/registry.py`

Проверка:
1. В event payload фиксируется выбранный режим.
2. Нельзя случайно запустить небезопасный режим.

## P2-02 [x] Реально задействовать provider_workers

Действия:
1. Вынести provider search в bounded pool.
2. Параллелить по провайдерам/запросам только в безопасных режимах.
3. Добавить backpressure по размеру очереди.

Артефакты:
- `pipeline/media.py`
- `pipeline/backpressure.py`

Проверка:
1. `provider_workers` влияет на throughput измеримо.
2. Нет взрывного роста памяти при большом количестве кандидатов.

## P2-03 [x] Реально задействовать download_workers

Действия:
1. Параллелить скачивание supporting/fallback изображений отдельным pool.
2. Ограничить одновременные скачивания через configurable лимит.
3. Сохранять порядок выбора assets независимо от порядка завершения скачивания.

Артефакты:
- `pipeline/media.py`

Проверка:
1. `download_workers` влияет на total run duration.
2. Результаты run детерминированы по выбранным asset IDs.

## P2-04 [x] Реально задействовать relevance_workers

Действия:
1. Вынести тяжелые операции relevance/ranking в отдельный bounded pool.
2. Ограничить очередь relevance и добавить graceful degradation при перегрузке.
3. Добавить метрики очереди: wait time, queue depth, dropped tasks.

Артефакты:
- `pipeline/media.py`

Проверка:
1. `relevance_workers` отражается в метриках и времени этапа.
2. При перегрузке run завершается контролируемо, без зависаний.

## P2-05 [x] Усилить early-stop стратегию

Действия:
1. Добавить threshold-based остановку после достижения достаточного качества.
2. Прекращать лишние provider queries после выполнения слотов.
3. Логировать причину ранней остановки.

Артефакты:
- `pipeline/media.py`

Проверка:
1. Среднее число запросов к провайдерам на абзац снижается.
2. Доля успешных подборов не ухудшается.

## P2-06 [x] Нормализовать timeout/retry политику по этапам

Действия:
1. Задать отдельные timeout для поиска, скачивания и relevance.
2. Привести retry budget к единой конфигурации.
3. Добавить fail-fast правила для фатальных ошибок Storyblocks.

Артефакты:
- `config/settings.py`
- `pipeline/media.py`
- `browser/storyblocks_backend.py`

Проверка:
1. Нет бесконтрольных долгих зависаний этапов.
2. Ошибки классифицируются предсказуемо.
