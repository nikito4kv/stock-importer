# Руководство пользователя Vid Img Downloader

## Что это за программа

`Vid Img Downloader` - это настольное приложение для подбора и скачивания медиа по нумерованному сценарию `.docx`.

Программа работает по абзацам:

1. импортирует сценарий;
2. анализирует каждый абзац;
3. подбирает видео и/или изображения;
4. скачивает найденные материалы;
5. сохраняет историю запусков и логи.

Главный источник видео - `Storyblocks`. Для изображений могут использоваться `Storyblocks`, `Pexels`, `Pixabay`, `Openverse`, `Wikimedia Commons` и, при отдельном разрешении, `Bing`.

## Важно перед началом

- Приложение ориентировано на Windows.
- Для работы режимов со `Storyblocks` нужен реальный аккаунт Storyblocks и установленный `Google Chrome` или `Microsoft Edge`.
- Сценарий должен быть в формате `.docx`.
- Основной рабочий путь для первого запуска - режим `Только видео Storyblocks`.

Если вы пользуетесь старыми заметками или скриншотами, имейте в виду: в текущей сборке актуальные вкладки называются `Главная`, `Ключи API`, `Сессия`, `Эксперт`, `История`.

## Что нужно подготовить заранее

### Обязательно

- `.docx`-сценарий с нумерованными абзацами;
- папку, куда будут сохраняться результаты;
- при работе со Storyblocks - установленный `Chrome` или `Edge`;
- при работе со Storyblocks - логин и пароль от аккаунта Storyblocks.

### Желательно

- `Gemini API key` - для анализа смысла абзацев и генерации запросов;
- `Pexels API key` - если хотите использовать Pexels;
- `Pixabay API key` - если хотите использовать Pixabay.

## Где брать API-ключи

### Gemini

Нужен для работы интеллектуальной части приложения.

- Официальная страница: `https://ai.google.dev/gemini-api/docs/api-key`
- Обычно путь такой:
  1. открыть страницу Gemini API;
  2. войти в Google-аккаунт;
  3. создать API key;
  4. скопировать ключ;
  5. вставить его во вкладке `Ключи API`.

Важно:

- в приложении проверяется только базовая корректность ключа;
- реальная проверка доступа происходит при первом обращении к Gemini.

### Pexels

- Официальная страница: `https://www.pexels.com/api/`
- Обычный путь:
  1. зарегистрироваться или войти в Pexels;
  2. открыть раздел API;
  3. получить личный API key;
  4. вставить его в строку `Pexels API key`.

Примечание: Pexels просит указывать источник и авторов там, где это возможно. Перед коммерческим использованием проверьте актуальные условия на сайте сервиса.

### Pixabay

- Официальная страница: `https://pixabay.com/api/docs/`
- Обычный путь:
  1. зарегистрироваться или войти в Pixabay;
  2. открыть страницу API;
  3. получить API key;
  4. вставить его в строку `Pixabay API key`.

Примечание: Pixabay отдельно пишет о лимитах запросов, кэшировании и запрете на массовые автоматические выгрузки без соблюдения правил сервиса. Перед активным использованием проверьте актуальные условия на сайте Pixabay.

### Какие источники не требуют ключа в текущем интерфейсе

- `Openverse`
- `Wikimedia Commons`
- `Bing`

Для них отдельные ключи в текущем GUI не вводятся.

### Storyblocks

Для Storyblocks в этом приложении нужен не API-ключ, а обычный вход через браузер во вкладке `Сессия`.

## Первый запуск

При первом старте приложение создает рабочую папку `workspace/` рядом со сборкой или в выбранном рабочем каталоге.

Внутри обычно появляются:

- `workspace/config` - настройки приложения;
- `workspace/projects` - импортированные проекты;
- `workspace/runs` - данные запусков;
- `workspace/presets` - сохраненные пресеты;
- `workspace/cache` - кеш поиска;
- `workspace/browser_profiles` - управляемые браузерные профили Storyblocks;
- `workspace/logs` - служебные логи;
- `workspace/secrets` - защищенно сохраненные ключи.

На Windows ключи сохраняются через системную защиту `DPAPI`, а не простым открытым текстом в интерфейсе.

## Верхняя панель: все кнопки

### `Открыть сценарий`

Импортирует `.docx`-сценарий в проект.

Как использовать:

1. сначала выбрать файл в поле `Файл сценария` или через кнопку `Обзор`;
2. затем нажать `Открыть сценарий`.

Что делает кнопка:

- читает `.docx`;
- создает локальный проект;
- проверяет нумерацию абзацев;
- подготавливает предварительный просмотр запуска.

Если нумерация плохая, приложение покажет предупреждение и не даст корректно стартовать запуск, пока сценарий не будет исправлен.

### `Старт`

Запускает новый run по текущему проекту и выбранным настройкам.

Перед запуском приложение проверяет:

- что сценарий загружен;
- что в нем есть нумерованные абзацы;
- что нет критичных ошибок нумерации;
- что для режима со Storyblocks готова сессия;
- что для free-image режима выбран хотя бы один бесплатный провайдер.

Если активный проект еще не открыт, приложение сначала попытается импортировать сценарий из поля `Файл сценария`, а затем стартовать run.

### `Продолжить`

Продолжает run, который был поставлен на паузу.

Используется, когда текущий запуск находится в состоянии `На паузе`.

### `Пауза`

Ставит запуск на кооперативную паузу.

Важно: это не мгновенная остановка. Приложение завершает текущий обрабатываемый абзац, после чего ставит run на паузу.

### `Остановить`

Отменяет активный запуск.

Важно: как и пауза, отмена происходит после завершения текущего абзаца. Уже скачанные файлы не удаляются автоматически.

### `Повторить ошибки`

Создает новый run только по тем абзацам, которые завершились с ошибкой в текущем запуске.

Если в run нет ошибочных абзацев, приложение покажет сообщение, что повторять нечего.

### `Проверить сессию`

Проверяет состояние авторизации Storyblocks.

Это дублирует одноименную кнопку из вкладки `Сессия`.

## Вкладка `Главная`

Это основной экран обычного пользователя.

### Блок `Что запускать`

#### Поле `Название проекта`

Необязательное поле.

- Если оставить пустым, приложение обычно подставит имя по названию файла сценария.
- Это имя нужно только для удобства в истории и внутри проекта.

#### Поле `Файл сценария`

Путь к `.docx`-сценарию.

- Можно вставить путь вручную.
- Можно нажать `Обзор` и выбрать файл через системное окно.

Ожидаемый формат:

- документ Word `.docx`;
- нумерованные абзацы;
- заголовки вроде `PART 1`, `CHAPTER 2`, `CONCLUSION` могут быть пропущены как служебные секции.

#### Кнопка `Обзор` рядом с полем сценария

Открывает окно выбора файла `.docx`.

После выбора:

- путь подставляется в поле;
- если поле `Название проекта` пустое, приложение часто подставляет имя файла как название проекта.

#### Поле `Папка вывода`

Папка, в которую будут складываться результаты конкретного запуска.

- Можно вставить путь вручную.
- Можно выбрать папку через кнопку `Обзор`.

#### Кнопка `Обзор` рядом с полем папки вывода

Открывает системное окно выбора папки.

#### Поле `Режим проекта`

Главный переключатель режима работы. От него зависит:

- нужны ли видео;
- нужны ли изображения Storyblocks;
- нужны ли бесплатные источники изображений;
- обязательна ли готовая Storyblocks-сессия.

Подробное описание всех режимов есть в разделе `Режимы проекта`.

### Блок `Перед запуском`

Это текстовый предварительный просмотр запуска.

Он показывает:

- название проекта;
- выбранный режим;
- папку вывода;
- сколько абзацев будет обрабатываться;
- список провайдеров;
- сводку по основным параметрам;
- предупреждения.

Типичные предупреждения:

- не задан ключ Gemini;
- Storyblocks-сессия не готова;
- для free-image режима не выбран ни один бесплатный провайдер;
- в сценарии проблемы с нумерацией.

Если блок пишет `Сначала откройте сценарий, чтобы увидеть параметры запуска`, значит файл еще не импортирован как проект.

### Блок `Что сейчас происходит`

Показывает живое состояние текущего запуска.

Здесь можно увидеть:

- сколько абзацев обработано;
- текущую стадию;
- какой абзац обрабатывается сейчас;
- примерную оценку оставшегося времени;
- краткую сводку `Найдено / Без результата / Ошибки`;
- общий процент выполнения.

Основные стадии:

- `Ожидание`
- `Загрузка`
- `Анализ`
- `Поиск`
- `Скачивание`
- `Оценка`
- `Сохранение`
- `Готово`

### Блок `Журнал событий`

Показывает подробный журнал обработки.

В строках журнала могут быть:

- время;
- уровень сообщения (`Инфо`, `Внимание`, `Ошибка`);
- стадия;
- текст события;
- контекст: номер абзаца, провайдер, запрос, ID ассета.

Это главный экран для диагностики, если run завис, что-то не скачалось или часть абзацев завершилась без результата.

#### Кнопка `Скачать логи`

Сохраняет текстовый отчет по текущему run в выбранный файл.

В экспорт входят:

- `Run ID`;
- `Project ID`;
- статус и стадия;
- время старта и завершения;
- краткая статистика;
- summary из manifest;
- полный список событий по run.

Если ни один run не выбран, кнопка недоступна.

## Вкладка `Ключи API`

Здесь сохраняются внешние ключи.

### Блок `Gemini`

#### Поле ввода Gemini key

- поле скрывает символы, как пароль;
- сюда вставляется ключ Gemini;
- пустой ключ сохранить нельзя;
- ключ с пробелами сохранить нельзя;
- слишком короткий ключ будет отвергнут.

#### Кнопка `Сохранить`

Сохраняет ключ Gemini в защищенное хранилище приложения.

После сохранения приложение показывает уведомление о том, что ключ выглядит структурно корректным.

#### Кнопка `Удалить`

Удаляет сохраненный ключ Gemini.

### Блок `Ключи стоков`

Содержит строки для провайдеров, которым нужны ключи.

#### `Pexels API key`

- поле для ключа Pexels;
- кнопка `Сохранить` сохраняет ключ;
- кнопка `Удалить` удаляет ключ.

Если ключ не задан, Pexels просто не будет реально использоваться при поиске.

#### `Pixabay API key`

- поле для ключа Pixabay;
- кнопка `Сохранить` сохраняет ключ;
- кнопка `Удалить` удаляет ключ.

Если ключ не задан, Pixabay будет пропущен.

## Вкладка `Сессия`

Эта вкладка нужна для авторизации в Storyblocks и проверки ее состояния.

### Информационная область

Показывает:

- `Профиль` - активный управляемый профиль приложения;
- `Состояние` - текущее здоровье сессии;
- `Аккаунт` - распознанный аккаунт Storyblocks, если вход выполнен;
- `Браузер готов` - может ли приложение работать с текущим браузерным профилем;
- `Окно входа открыто` - открыт ли браузер для логина;
- `Текущий URL` - адрес, на котором сейчас находится браузер Storyblocks.

Иногда дополнительно могут появляться:

- `Debug port`;
- сведения об импорте профиля;
- сообщение о ручном подтверждении;
- последняя ошибка.

Для обычного пользователя это служебные строки. Если не понимаете их смысл, можно просто ориентироваться по полю `Состояние`.

### Значения поля `Состояние`

- `Неизвестно` - приложение еще не проверяло сессию или не может уверенно определить ее состояние;
- `Готово` - можно запускать режимы, которым нужен Storyblocks;
- `Нужен вход` - нужно войти в аккаунт через браузер;
- `Требуется проверка` - Storyblocks просит дополнительное подтверждение, капчу или промежуточный шаг;
- `Сессия истекла` - авторизация устарела, нужен новый вход;
- `Доступ ограничен` - Storyblocks ограничил доступ или не дает пройти автоматизации.

### Кнопка `Войти в браузере`

Открывает управляемое окно Chrome или Edge для входа в Storyblocks.

Правильный порядок:

1. нажать `Войти в браузере`;
2. выполнить вход в Storyblocks в открытом окне;
3. не закрывать окно сразу после входа;
4. вернуться в приложение;
5. нажать `Проверить сессию`.

### Кнопка `Проверить сессию`

Проверяет, готова ли текущая Storyblocks-сессия к автоматической работе.

Именно после этой проверки состояние должно перейти в `Готово`.

### Кнопка `Открыть браузер Storyblocks`

Просто открывает управляемое окно Storyblocks в текущем профиле.

Полезно, если нужно:

- визуально проверить, открыт ли нужный аккаунт;
- оставить браузер доступным перед запуском;
- руками убедиться, что сайт работает нормально.

### Кнопка `Выйти`

Сбрасывает текущую авторизацию Storyblocks в приложении.

После этого обычно потребуется новый вход.

### Кнопка `Сменить аккаунт`

Создает новый цикл входа для другого Storyblocks-аккаунта.

По сути это способ начать работу с чистого профиля приложения для другой учетной записи.

### Важное ограничение

Во время активного запуска кнопки управления Storyblocks-сессией блокируются. Это сделано специально, чтобы пользователь не ломал сессию посреди скачивания.

## Вкладка `Эксперт`

Это вкладка продвинутых настроек. Если не уверены, что меняете, оставляйте значения по умолчанию.

### Блок `Поведение запуска`

#### Поле `Строгость`

Варианты:

- `Простой`
- `Баланс`
- `Строгий`

Для большинства случаев используйте `Баланс`.

Практический смысл:

- `Простой` - более мягкий и общий подход;
- `Баланс` - основной безопасный режим для обычной работы;
- `Строгий` - более узкая и требовательная логика отбора.

Если вы не тестируете разные сценарии качества, не меняйте это поле без необходимости.

#### Флажок `Медленный режим браузера`

Замедляет поведение браузерной автоматизации.

Когда полезно:

- Storyblocks загружается нестабильно;
- браузер не успевает отрисовать страницу;
- логин или скачивание иногда срываются.

Когда можно выключить:

- если у вас стабильный интернет;
- сайт работает быстро;
- хотите немного ускорить работу.

Для первого запуска лучше оставить включенным.

#### Флажок `Разрешить обычный веб-поиск изображений`

Разрешает использовать `Bing` как источник обычных веб-изображений.

Это самый осторожный флажок во всей вкладке, потому что:

- источник более шумный;
- релевантность может быть хуже;
- вопросы лицензий менее предсказуемы, чем у стоков и открытых библиотек.

Если вам важнее аккуратность и понятность источников, оставьте выключенным.

### Блок `Источники бесплатных изображений`

Этот блок используется только в режимах, где разрешены бесплатные изображения.

#### Флажок `Pexels`

Включает поиск по Pexels.

Чтобы он реально работал, нужно сохранить `Pexels API key` во вкладке `Ключи API`.

#### Флажок `Pixabay`

Включает поиск по Pixabay.

Чтобы он реально работал, нужен `Pixabay API key`.

#### Флажок `Openverse`

Включает поиск по Openverse.

Отдельный API key в текущем интерфейсе не нужен.

#### Флажок `Wikimedia Commons`

Включает поиск по Wikimedia Commons.

Отдельный API key в текущем интерфейсе не нужен.

#### Флажок `Bing (обычные веб-изображения)`

Включает Bing в список бесплатных источников, но фактически он учитывается только если одновременно включен флажок `Разрешить обычный веб-поиск изображений`.

То есть для Bing нужны сразу два условия:

1. включить сам `Bing` в списке провайдеров;
2. включить `Разрешить обычный веб-поиск изображений`.

### Блок `Пресеты`

Пресеты - это сохраненные наборы настроек.

#### Поле имени пресета

Сюда вводится название пресета, например:

- `Только видео Storyblocks`
- `Видео плюс бесплатные картинки`
- `Free images only`

#### Кнопка `Сохранить`

Сохраняет текущую комбинацию параметров как пресет.

Если имя пустое, сохранить нельзя.

#### Кнопка `Загрузить`

Загружает пресет с именем из поля имени и применяет его настройки к интерфейсу.

Важно: имя нужно ввести точно так, как оно было сохранено.

#### Кнопка `Экспорт`

Сохраняет выбранный пресет в `.json`-файл, чтобы перенести его на другой компьютер или сделать резервную копию.

#### Кнопка `Импорт`

Импортирует пресет из `.json`-файла в приложение.

Важно: после импорта пресет сохраняется в список пресетов, а его имя подставляется в поле. Чтобы реально применить его настройки к форме, нажмите `Загрузить`.

### Блок `Технические параметры`

Если вы не занимаетесь диагностикой и настройкой производительности, оставьте эти значения по умолчанию.

#### `Потоки абзацев`

Сколько абзацев может обрабатываться параллельно на верхнем уровне.

Больше значение - выше нагрузка и выше риск нестабильности на слабом ПК или плохой сети.

#### `Потоки провайдеров`

Сколько одновременных запросов к источникам может выполняться при подборе.

#### `Потоки скачивания`

Сколько файлов можно скачивать параллельно.

#### `Потоки релевантности`

Сколько задач оценки релевантности можно выполнять параллельно.

#### `Размер очереди`

Внутренний лимит очереди задач. Обычному пользователю обычно не нужно менять это поле.

#### `Таймаут запуска, мс`

Сколько приложение ждет запуска браузера.

#### `Таймаут навигации, мс`

Сколько приложение ждет открытия страницы или перехода по сайту.

#### `Таймаут скачивания, с`

Сколько секунд ждать скачивания одного файла.

Если у вас медленный интернет и скачивание часто рвется по времени, можно увеличить это значение.

#### `Top-K релевантности`

Сколько кандидатов оставлять до этапа финальной оценки релевантности.

Для обычной работы менять не нужно.

#### `Лимит повторов`

Сколько раз система может повторить неудачную попытку в ряде шагов.

#### `Папка кэша`

Техническое поле, связанное с кешем провайдеров.

В обычной работе лучше не менять.

#### `Путь к профилю браузера`

Техническое поле, связанное с профилями Storyblocks.

В обычной работе лучше не менять и оставить значение по умолчанию.

## Вкладка `История`

Здесь хранится список прошлых запусков.

Каждая строка содержит:

- `run_id`;
- статус;
- дату создания.

Что можно делать:

- выбрать run из истории;
- вернуть его в активный контекст приложения;
- затем посмотреть прогресс и журнал на вкладке `Главная`;
- при необходимости сохранить лог через `Скачать логи`.

При выборе записи история делает этот run текущим и обычно возвращает вас на вкладку `Главная`.

## Переключатель темы

В правом верхнем углу окна есть поле `Тема`.

Доступны варианты:

- `Темная`
- `Светлая`

После переключения тема сохраняется и должна восстановиться после следующего запуска приложения.

## Режимы проекта

### `Только видео Storyblocks`

- используются только видео Storyblocks;
- изображения не ищутся;
- нужен вход в Storyblocks;
- это лучший режим для первого успешного запуска.

### `Видео Storyblocks + изображения Storyblocks`

- видео ищутся в Storyblocks;
- изображения тоже ищутся в Storyblocks;
- нужен вход в Storyblocks.

### `Видео Storyblocks + бесплатные изображения`

- видео берутся из Storyblocks;
- изображения ищутся в бесплатных источниках;
- нужен вход в Storyblocks;
- обязательно должен быть выбран хотя бы один бесплатный провайдер.

### `Изображения Storyblocks + бесплатные изображения`

- проект без видеослотов;
- изображения сначала ищутся в Storyblocks;
- бесплатные источники используются как запасной вариант;
- нужен вход в Storyblocks.

### `Только бесплатные изображения`

- видео не используются;
- Storyblocks не нужен;
- изображения ищутся только в бесплатных источниках;
- необходимо включить хотя бы один бесплатный провайдер.

## Рекомендуемый порядок работы

### Первый запуск с Storyblocks

1. открыть вкладку `Ключи API` и сохранить ключ `Gemini`;
2. при необходимости сохранить `Pexels API key` и `Pixabay API key`;
3. открыть вкладку `Сессия`;
4. нажать `Войти в браузере`;
5. войти в Storyblocks в открывшемся окне Chrome или Edge;
6. не закрывать это окно;
7. нажать `Проверить сессию` и дождаться статуса `Готово`;
8. перейти на вкладку `Главная`;
9. выбрать `.docx`-сценарий;
10. выбрать папку вывода;
11. выбрать режим `Только видео Storyblocks`;
12. нажать `Открыть сценарий`;
13. проверить блок `Перед запуском`;
14. нажать `Старт`.

### Если Storyblocks не нужен

1. сохранить ключ `Gemini`;
2. при желании сохранить ключи `Pexels` и `Pixabay`;
3. на вкладке `Главная` выбрать сценарий и папку вывода;
4. на вкладке `Эксперт` включить нужные бесплатные провайдеры;
5. вернуться на `Главная`;
6. выбрать режим `Только бесплатные изображения`;
7. импортировать сценарий;
8. нажать `Старт`.

## Куда сохраняются результаты

Если вы указали собственную папку вывода, структура обычно будет такой:

- `<ваша папка>/<run_id>/downloads/videos/` - скачанные видео текущего запуска;
- `<ваша папка>/<run_id>/downloads/images/` - скачанные изображения текущего запуска.

Дополнительно технические данные запуска сохраняются внутри `workspace/runs/<run_id>/`.

Там обычно лежат:

- `run.json` - состояние запуска;
- `manifest.json` - итоговая структура выбранных медиа и служебная информация.

Общий служебный лог приложения пишется в `workspace/logs/app.log`.

## Типичные сообщения и что они значат

### `Исправьте нумерацию сценария перед запуском`

В `.docx` нарушены номера абзацев.

Что делать:

- исправить нумерацию в Word;
- снова импортировать сценарий.

### `Для выбранного режима нужна готовая сессия Storyblocks`

Вы выбрали режим со Storyblocks, но еще не прошли вход.

Что делать:

- открыть вкладку `Сессия`;
- выполнить вход;
- проверить сессию до состояния `Готово`.

### `Выберите хотя бы один источник бесплатных изображений для этого режима`

Вы выбрали режим с бесплатными изображениями, но отключили все бесплатные провайдеры.

Что делать:

- перейти во вкладку `Эксперт`;
- включить хотя бы один источник: `Pexels`, `Pixabay`, `Openverse`, `Wikimedia Commons`, а при необходимости и `Bing`.

### Ключ Gemini сохраняется, но потом появляются ошибки API

Это нормально: приложение при сохранении проверяет только базовую структуру ключа.

Что делать:

- проверить, что ключ действителен;
- проверить лимиты и доступ в Google;
- повторить запуск позже, если исчерпана квота.

### Storyblocks вошел в аккаунт, но статус все еще не `Готово`

Что делать:

- немного подождать, пока сайт закончит редиректы;
- убедиться, что окно браузера не закрыто;
- нажать `Проверить сессию` еще раз.

### Во время run нельзя нажимать кнопки сессии

Это штатное поведение. Приложение специально блокирует изменение Storyblocks-сессии во время активного запуска.

## Что лучше не делать

- не закрывать управляемый браузер Storyblocks во время активного run;
- не использовать один и тот же профиль Storyblocks параллельно в нескольких процессах;
- не включать `Bing`, если вам важнее предсказуемость лицензий и качества;
- не менять технические пути и потоки без необходимости;
- не стартовать проект с невалидной нумерацией сценария.

## Что есть в проекте, но не показано в текущем релизном GUI

В кодовой базе есть следы более широких функций: импорт внешней браузерной сессии, ручные подтверждения, workbench по абзацам, ручная фиксация ассетов и точечные rerun по абзацам.

Но в текущем пользовательском окне эти функции обычному пользователю не показаны. Поэтому если вы их не видите в интерфейсе, это нормально: ориентируйтесь только на элементы, описанные в этом руководстве.
